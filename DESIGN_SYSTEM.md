# ClearMark Design System

Version: `v1.2 Target`

Status: `Archived Draft`

ClearMark is an Apple-inspired icon system for network utility and traffic dispatch profiles.

## Project

ClearMark Icon Set

## Target Platforms

- Surge
- Stash
- Clash Verge
- Mihomo Party
- iOS Network Utilities

## Design Philosophy

ClearMark should feel like a unified operating system component library rather than a collection of unrelated icons.

Keywords:

- Premium
- Minimal
- Consistent
- Frosted Glass
- Layered
- Calm
- Professional

Priority:

1. Visual Consistency
2. Brand Recognition
3. Decoration

## Canvas

- Master source: `1024 x 1024 px`
- Release export: `512 x 512 px PNG`
- Background: transparent

## Safe Area

- Preferred safe area: `860 x 860 px`
- Allowed range: `840 x 840 px` to `900 x 900 px`

Icons should use more canvas space than standard mobile app icons because they are primarily displayed in compact Surge, Stash, and Clash policy-group lists.

Primary graphics should remain inside the safe area.

## Visual Weight

This is the most important rule.

Do not normalize geometric size. Normalize visual weight.

- Target visual weight: `60%`
- Allowed range: `58%` to `65%`
- Maximum deviation: `+/-5%`

All icons should appear visually equal when displayed in Surge policy groups, Stash policy groups, and Clash Verge lists.

Visual consistency is more important than geometric consistency.

## Frosted Glass System

Style: iOS 26 Light Frosted Glass

Characteristics:

- Soft translucency
- Subtle internal glow
- Very soft shadow
- Premium material feeling
- Clean edges

Corner radius:

- `22%` to `25%`

Avoid heavy shadows and aggressive gradients.

## Color System

Avoid excessive color diversity. The icon set should feel like one ecosystem.

Color ratio:

- Neutral base: `80%`
- Accent colors: `20%`

## Neutral Palette

Primary colors:

- `#F5F7FA`
- `#E8EDF2`
- `#D6DDE5`
- `#C4CDD8`
- `#AAB4C0`

Used for:

- Backgrounds
- Most icons
- Region icons
- Infrastructure icons

## Accent Palette

Maximum accent categories: `6`

- AI: Purple
- WW: Green
- Direct: Blue
- China: Red
- Reject: Orange
- Media: Cyan

No additional accent categories unless necessary.

## Strict Color Rules

Do not use:

- National flag colors
- Full brand colors
- Rainbow palette

Examples:

- US icon should not be red, white, and blue.
- JP icon should not be red and white.
- UK icon should not use Union Jack colors.
- Spotify should not use bright Spotify green.
- Netflix should not use bright Netflix red.

Use monochrome logo treatment with ClearMark color language instead.

## Region Icons

Examples:

- HK
- TW
- JP
- KR
- US
- UK
- SG
- CH
- NL
- AT

Requirements:

- Same visual weight
- Same glass style
- Same color language

Target visual weight:

- `60%` to `62%`

All region icons should feel like siblings.

## AI Icons

Examples:

- AI
- ChatGPT
- Claude
- Gemini

Color:

- Purple accent

Target visual weight:

- `62%` to `65%`

AI icons may be slightly larger than standard icons for fast recognition.

## Media Icons

Examples:

- Netflix
- Disney
- Spotify
- YouTube
- Prime Video
- HBO

Style:

- Monochrome logo
- Cyan glass base

Target visual weight:

- `60%` to `62%`

## Infrastructure Icons

Examples:

- Direct
- Proxy
- Server
- Premium
- Reject

These icons define the baseline.

Target visual weight:

- `60%` to `62%`

## Worldwide Icon

Replace world map silhouettes because they become visually weak below `48px`.

Use a green frosted globe.

Requirements:

- Simple continents
- No detailed projections
- High recognizability
- Balanced visual weight

Target visual weight:

- `62%`

This becomes the official WW icon language.

## Map Series

Avoid:

- EU_Map
- Asia_Map
- Africa_Map
- North_America_Map
- South_America_Map

Reason:

- Poor visibility at small sizes

Replace with:

- Globe-based symbols
- Region-themed minimal badges

## Brand Icon Normalization

Current small-logo issues should be corrected.

Examples:

- Hulu
- ESPN
- Claude
- WhatsApp
- Twitter
- Telegram

Increase visual weight by `10%` to `20%` until they match the baseline icons.

Target visual weight:

- `60%` to `62%`

## Quality Control

Every icon must pass:

1. 32px visibility test
2. 48px visibility test
3. Surge list test
4. Stash list test
5. Light mode test
6. Dark mode test

Success criteria:

- No icon should appear noticeably larger or smaller than adjacent icons.
- No icon should visually break the ClearMark ecosystem.

## Target Result

The final icon set should resemble a premium Apple-designed network utility ecosystem.

Users should perceive:

- Clean
- Consistent
- Professional
- High-end
- Calm
- Cohesive

Users should not perceive:

- Colorful
- Brand-heavy
- Mixed-style
- Inconsistent

The entire icon library should look like it was designed by one team, at one time, under one design language.
